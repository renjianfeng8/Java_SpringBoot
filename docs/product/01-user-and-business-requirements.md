# 用户需求与业务需求

## 1. 背景

传统影院售票依赖线下窗口或分散的人工排片管理，用户需要到店查询影片、场次和座位，影院也难以及时统计订单、维护影片和管理排片。当前系统通过 Web 平台整合电影信息、影院信息、排片、选座、订单和评价，形成从“影片上架”到“用户购票”的闭环。

本项目已实现三端角色分离：管理员后台、影院端、用户端。需求文档以当前系统能力为基线，不虚构未实现的大型商业支付、退票、会员积分等能力。

## 2. 用户画像

### 2.1 普通用户

- 目标：快速找到想看的电影和合适影院，完成选座购票。
- 高频场景：浏览热映电影、搜索电影、查看影院详情、选择场次、在线选座、查看订单。
- 痛点：
  - 不想线下排队查询影讯。
  - 想提前知道场次、价格和座位。
  - 想比较电影评分、票房和基本信息。
  - 希望订单状态清晰可追踪。

### 2.2 影院管理员

- 目标：维护本影院基础信息、影厅、排片和订单。
- 高频场景：新增影厅、创建排片、查看订单、维护影院资料。
- 痛点：
  - 排片信息分散，人工维护容易出错。
  - 订单管理缺少统一后台。
  - 影院资料和影厅信息需要可维护。

### 2.3 系统管理员

- 目标：管理平台基础数据，保障系统内容和运营秩序。
- 高频场景：审核影院、维护影片、分类、地区、演员、公告、视频，查看订单和评价。
- 痛点：
  - 多角色数据需要统一管理。
  - 影片、影院、排片、订单之间关系复杂，需要平台级视角。
  - 需要避免普通用户或影院越权访问管理资源。

## 4. 业务需求

### 4.1 业务目标

- 提供一个覆盖影片、影院、排片、选座、订单、评价的完整票务管理闭环。
- 降低影院排片和订单管理成本。
- 提升用户购票体验，减少线下排队和信息不透明。
- 通过管理后台统一维护平台数据，保证内容、权限和基础配置可控。

### 4.2 核心业务流程

1. 平台管理员维护电影、分类、地区、演员、预告片等基础数据。
2. 平台管理员审核影院账号和影院资料。
3. 影院管理员维护影厅，并为影片创建排片。
4. 用户浏览电影或影院，选择排片。
5. 用户进入选座页，选择座位并提交订单。
6. 系统生成订单，记录用户、电影、影院、影厅、场次、座位和金额。
7. 用户查看订单，影院和管理员可在后台查看订单。
8. 用户观影后可以评价电影，评分进入排行榜或详情展示。

## 5. 权限需求

| 角色 | 可访问端 | 核心权限 |
|------|----------|----------|
| USER | `/front/*` | 浏览电影、影院、排片，选座购票，查看个人订单和资料 |
| CINEMA | `/back/*` | 维护本影院资料、影厅、排片、订单 |
| ADMIN | `/manage/*` | 管理平台全部基础数据、影院审核、订单和评价 |

> 角色详细定义见[术语表 §1](./glossary.md#1-角色与权限)，路由和接口权限实现见 [PRD §4](./03-product-requirements-document.md#4-角色与权限)。

安全要求：

- 未登录用户不能访问需要认证的页面和接口。
- 用户不能访问影院后台或管理后台。
- 影院账号不能访问管理后台。
- 管理端资源需要管理员权限。
- 密码修改以 JWT 中的角色和用户 ID 为准，不信任前端传入角色。

## 6. 非功能需求

| 类型 | 需求 |
|------|------|
| 可用性 | 页面路径清晰，三端导航明确，常用操作支持分页、搜索、重置 |
| 安全性 | JWT 认证、角色守卫、后端拦截器、密码加密、上传 MIME 白名单 |
| 可维护性 | 后端使用 BaseController/BaseService/BaseMapper 降低 CRUD 重复代码（详见 [PRD §8.2](./03-product-requirements-document.md#82-实体关系图-erd) 实体关系设计） |
| 数据一致性 | 电影、影院、影厅、排片、订单之间使用 ID 关系和外键约束（详见 [术语表 §2](./glossary.md#2-核心业务实体)） |
| 可验证性 | 支持 Maven 编译、前端构建、数据库初始化、Playwright E2E（详见 [PRD §11.2](./03-product-requirements-document.md#112-技术验收)） |
| 可扩展性 | 后续可扩展支付、退票、会员、优惠券、座位模板、数据看板 |

## 7. 需求边界

当前版本不包含：

- 第三方支付真实接入。
- 真实出票码、验票机或影院硬件对接。
- 退票、改签和复杂售后流程。
- 动态票价、优惠券、会员积分。
- 多城市定位、地图导航和商业推荐算法。
- 高并发座位锁定和支付超时自动释放。

这些能力已规划为后续迭代方向，详见 [PRD 优化路线图 §3 (P1)](./04-prd-optimization-roadmap.md#3-p1建议下一阶段优先优化) 和 [§4 (P2)](./04-prd-optimization-roadmap.md#4-p2商业化增强能力)。
